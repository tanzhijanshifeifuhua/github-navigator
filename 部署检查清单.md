- [ ] 初始化 Git 仓库
  ```bash
  git init
  ```

- [ ] 添加 .gitignore 文件（已创建）

- [ ] 提交所有文件
  ```bash
  git add .
  git commit -m "Initial commit: GitHub 项目中文分类导航"
  ```

- [ ] 创建 GitHub 仓库

- [ ] 推送到 GitHub
  ```bash
  git remote add origin https://github.com/你的用户名/你的仓库.git
  git push -u origin main
  ```

## 选择部署方式

### 方式 A: Vercel（推荐新手）

- [ ] 访问 https://vercel.com
- [ ] 使用 GitHub 账号登录
- [ ] 导入 GitHub 仓库
- [ ] 点击 Deploy
- [ ] 等待部署完成
- [ ] 测试访问地址
- [ ] （可选）绑定自定义域名

**预计时间**: 5 分钟

### 方式 B: Docker（推荐有服务器）

- [ ] 安装 Docker 和 Docker Compose
- [ ] 克隆仓库到服务器
- [ ] 运行部署脚本
  ```bash
  chmod +x deploy.sh
  ./deploy.sh
  # 选择选项 1
  ```
- [ ] 测试访问 http://服务器IP:5000
- [ ] （可选）配置 Nginx 反向代理
- [ ] （可选）配置 SSL 证书

**预计时间**: 15 分钟

### 方式 C: Railway

- [ ] 访问 https://railway.app
- [ ] 连接 GitHub 仓库
- [ ] 等待自动部署
- [ ] 生成域名
- [ ] 测试访问

**预计时间**: 5 分钟

### 方式 D: Linux 服务器

- [ ] 准备 VPS 服务器
- [ ] 安装必要软件（Python, Nginx）
- [ ] 上传代码
- [ ] 配置虚拟环境
- [ ] 配置 Systemd 服务
- [ ] 配置 Nginx
- [ ] 配置 SSL（Let's Encrypt）
- [ ] 测试访问

**预计时间**: 30-60 分钟

## 快速命令参考

### 查看服务状态
```bash
# Docker
docker-compose ps

# Systemd
sudo systemctl status github-navigator
```

### 查看日志
```bash
# Docker
docker-compose logs -f

# Systemd
sudo journalctl -u github-navigator -f
```

### 重启服务
```bash
# Docker
docker-compose restart

# Systemd
sudo systemctl restart github-navigator
```

### 更新数据
```bash
python crawler.py
python add_chinese_descriptions.py
```
